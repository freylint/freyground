Suck Less, Self Less, and the Trailing Edge

I've been attempting to understand software stability for many years. For about almost 5 years, I've been using correctness as primary principle driving engineering decisions. This drove decisions to use technologies like Rust, NixOS, and Ansible. On the surface this suggests an enthusiasm for functional programming and declarativeness. After almost 5 years using these technologies for the creation of the Freyground project, I have determined that they ultimately have limited utility compared to the legacy systems that I eschewed in the name of correctness. Rust is a systems programming language that's extremely flexible until the layers of abstraction common in its users library design become too burdensome to work around. NixOS provides the promise of a reproducible build system, but fails due to lack of general adoption and platform support. If nix and rpm/apt were in opposite places, we would be in a better world. We are in the world where reproducibility is done with domain specific tools that have become a requirement for cooperation in the wider software ecosystem. Ansible has it's place. That place is not a homelab. It makes the promise of declarative configuration management. It fails to meet up to this promise while making it harder to work with the underlying system due to the rigidity of the way it handles shell commands. Modular composable shell calls in the form of:
ssh -c user@host 'sh -c "foo"'
is better handled by general purpose tools like Make and Python in the homelab.
